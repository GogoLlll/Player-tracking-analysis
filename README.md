# Система трекинга и анализа взаимодействия игроков на футбольном поле

Программа для автоматического обнаружения, отслеживания
и анализа действий футболистов по видеозаписи матча, разработанная
в рамках выполнения дипломной работы в филиале МГУ имени
М.В. Ломоносова в г. Ереване на факультете прикладной математики и информатики.


Проект включает графический интерфейс (PyQt6) с живым просмотром, а также
консольные скрипты для пакетной обработки.

> Ссылка на [дипломную работу.](https://disk.360.yandex.ru/d/Ys_AFlFnWf5VzA)

## Возможности

- **Детекция игроков** на каждом кадре.
- **Многообъектный трекинг** на базе BoT-SORT с внешним Re-ID.
- **Оффлайн пост-обработка треков**: глобальная склейка фрагментов одного
  игрока, банк Re-ID-эмбеддингов для «воскрешения»
  потерянных ID, сглаживание боксов на пропусках детекции.
- **Классификация по командам** по цвету формы,
  с выделением вратарей и арбитра.
- **Калибровка поля (гомография)** через PnLCalib: перевод пиксельных
  координат в метрические координаты плана поля.
- **Мини-карта**: проекция игроков на схему поля в реальном времени.
- **Оценка позы** игроков со сглаживанием ключевых точек.
- **Детекция нарушений**: правиловая логика и ML-классификатор.
- **Аналитика матча**: пройденная дистанция, средняя/максимальная скорость,
  покрытие позами, агрегаты по командам, список событий.
- **Графический интерфейс**: выбор видео и моделей, переключатели
  анализа, живой просмотр, окно мини-карты, вкладка аналитики с экспортом.

## Установка

### 1. Установить зависимости проекта

```bash
pip install -r requirements.txt
```

> Для GPU установите соответствующую сборку `torch`/`torchvision` по
> инструкции с [pytorch.org](https://pytorch.org) до или вместо строк из
> `requirements.txt`.

### 2. Клонировать репозиторий калибровки поля

Гомография использует код **PnLCalib**. Репозиторий не входит в проект и
клонируется рядом с ним:

```bash
git clone https://github.com/mguti97/PnLCalib.git
```

> **Не запускайте** `pip install -r PnLCalib/requirements.txt` - это пиновый
> freeze всего окружения (`numpy==2.0.2`, `PyQt5`, cuda-колёса), который
> конфликтует с этим кодом. Все необходимые для нашего пути зависимости
> PnLCalib (включая `shapely` и `matplotlib`) уже есть в нашем
> `requirements.txt`.  LightGlue для генерации гомографии не требуется.

Скачайте веса PnLCalib (детектор ключевых точек и линий поля) согласно
инструкции в его README - они понадобятся при
генерации гомографии.

### 3. Скачать веса моделей

См. раздел [Модели и веса](#модели-и-веса).

## Модели и веса

Веса (`*.pt`) не хранятся в репозитории и кладутся в
папку `models/` (детекция/поза) и в корень (Re-ID):

| Файл | Назначение | Источник |
|------|------------|----------|
| `models/yolo12m.pt` (или иной YOLO-детектор) | детекция игроков | Ultralytics |
| `models/yolo11l-pose.pt` (или иная YOLO-pose) | оценка позы | Ultralytics |
| `osnet_x1_0_msmt17.pt` | Re-ID-эмбеддинги (OSNet) | BoxMOT / torchreid |
| веса PnLCalib (keypoints + lines) | калибровка поля | репозиторий PnLCalib |

Имена детектора и pose-модели по умолчанию заданы в `RunConfig`
(`pipeline_engine.py`) и переопределяются в графическом интерфейсе.

## Калибровка поля (гомография)

Гомография переводит пиксельные координаты в метрические координаты плана
поля. Её удобно посчитать один раз для видео и закэшировать в
`homography.npz`.

**Из графического интерфейса (основной способ):** в блоке мини-карты/
гомографии указать репозиторий PnLCalib и веса, выбрать видео и запустить
генерацию.

**Из консоли (опционально, для пакетных сценариев):**

```bash
python precompute_homography.py path/to/match.mp4 \
    --pnlcalib-repo ./PnLCalib \
    --kp-weights   path/to/keypoints.pt \
    --line-weights path/to/lines.pt \
    --output       homography.npz \
    --keyframe-interval 30 \
    --device cuda:0
```

Полный список флагов - `python precompute_homography.py --help`.

Гомография считается на опорных кадрах и интерполируется между ними; без
неё трекинг и поза работают, но мини-карта и метрическая аналитика
(дистанции/скорости) будут недоступны.

## Запуск

### Графический интерфейс (основной сценарий)

```bash
python gui_app.py
```

В интерфейсе доступны: выбор видео, выбор YOLO-детектора и pose-модели,
переключатели анализа (поза/нарушения/команды/мини-карта/пост-обработка),
выбор устройства, живой просмотр с окном мини-карты и активных треков,
а также вкладка «Аналитика матча» с возможностью экспорта.

На выходе сохраняются два видео (обычное и merged) и файлы аналитики.

### Консольная пакетная обработка

```bash
python batch_process.py
```

Скрипт выполняет полный конвейер и сохраняет результаты. Параметры
задаются внутри скрипта/через аргументы (`--help`).

## Выходные данные

- **`*_tracked.mp4`** - видео онлайн-трекинга (боксы, ID, команды, поза,
  оверлеи нарушений, при желании - вшитая мини-карта).
- **`*_merged.mp4`** - видео после пост-обработки: склеенные ID, сглаженные
  боксы, заполненные пропуски детекции.
- **Аналитика (JSON/CSV)** - по каждому игроку: команда, число кадров,
  время на поле, дистанция (м), средняя и максимальная скорость (м/с),
  покрытие позами; агрегаты по командам; список событий-нарушений;
  системные метрики качества (покрытие гомографией, число ID до/после
  склейки и т. д.).

## Конфигурация и параметры

Все параметры конвейера собраны в датаклассе `RunConfig`
(`pipeline_engine.py`): пути к моделям и видео, пороги детекции/IoU,
переключатели анализа, параметры позы, нарушений, гомографии, мини-карты,
трекинга и склейки. Графический интерфейс формирует `RunConfig` из
выбранных пользователем настроек; для CLI параметры задаются в коде скрипта
и через аргументы командной строки.

## Возможные проблемы

- **Нет мини-карты / нулевые дистанции** - не сгенерирована гомография
  (`homography.npz`) или PnLCalib/веса указаны неверно.
- **`ModuleNotFoundError` при гомографии** - не клонированы `PnLCalib`/
  `LightGlue` или не установлены их зависимости.
- **Работает медленно** - обработка идёт на CPU; установите CUDA-сборку
  PyTorch и выберите устройство `cuda:0`.
